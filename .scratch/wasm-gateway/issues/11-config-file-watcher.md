# 11 — Config 파일 감시 + 자동 reload

## What to build

`gw-admin start` 실행 중 `gateway/config/` 디렉토리를 감시하다가 파일 변경이 감지되면 nginx.conf를 재생성하고 `nginx -s reload`를 자동 실행한다. `notify` 크레이트를 사용한다.

동작 시나리오: `gw-admin start` → 정책 JSON 파일 편집 → 자동으로 nginx reload → 새 설정 즉시 반영.

## Acceptance criteria

- [ ] `gw-admin start` 실행 중 `gateway/config/` 내 파일 변경 시 자동으로 nginx.conf 재생성 + reload가 일어난다
- [ ] reload 중 진행 중인 요청이 끊기지 않는다 (nginx graceful reload)
- [ ] 잘못된 config 파일로 변경 시 reload를 건너뛰고 에러 로그를 남긴다 (기존 nginx 설정 유지)
- [ ] `gw-admin reload` 커맨드로 수동 reload도 가능하다

## Agent Brief

**Category:** enhancement
**Summary:** `gw-admin start` 실행 중 `gateway/config/` 변경을 감지하면 nginx.conf를 재생성하고 `nginx -s reload`를 자동 실행한다

**Current behavior:**
`gw-admin start`가 nginx를 기동하지만 config 파일 변경을 감시하지 않는다. 설정을 바꾸려면 `gw-admin reload`를 수동으로 실행해야 한다.

**Desired behavior:**
`gw-admin start`가 nginx를 기동한 뒤 백그라운드 스레드에서 `gateway/config/` 디렉토리를 감시한다. 파일이 생성/수정/삭제되면 새 config를 파싱하고, 유효하면 nginx.conf를 재생성 후 `nginx -s reload`를 실행한다. 파싱 실패 시 reload를 건너뛰고 에러 로그를 출력하며 기존 nginx 설정을 유지한다. `notify` 크레이트를 사용한다.

**Key interfaces:**
- `notify::RecommendedWatcher` — `gateway/config/` 재귀 감시
- `gw-admin reload` — 수동 reload 커맨드 (파일 감시와 독립적으로 동작)
- reload 흐름: 파일 변경 이벤트 → config 파싱 → 파싱 성공 시 nginx.conf 재생성 → `nginx -s reload` 실행

**Acceptance criteria:**
- [ ] `gw-admin start` 실행 중 policy JSON 파일 수정 시 자동으로 nginx reload가 일어난다
- [ ] reload 중 진행 중인 요청이 끊기지 않는다 (nginx graceful reload)
- [ ] 잘못된 JSON으로 파일을 변경 시 reload를 건너뛰고 에러 메시지를 출력하며 기존 설정을 유지한다
- [ ] `gw-admin reload` 커맨드로 수동 reload가 동작한다
- [ ] 자동 reload 후 변경된 설정이 새 요청에 즉시 반영된다

**Out of scope:**
- Admin REST API를 통한 동적 config 변경
- reload 없이 실행 중인 필터에 config를 핫패치하는 기능
