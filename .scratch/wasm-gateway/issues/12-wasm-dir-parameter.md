# 12 — `--wasm-dir` 인자 추가 + template 파라미터화

## What to build

`gw-admin`이 nginx.conf를 생성할 때 Wasm 모듈 경로를 `--wasm-dir` 인자로 제어할 수 있도록 한다.

현재 `template.rs`는 Wasm 경로를 `../../target/wasm32-wasip1/wasm-release`로 하드코딩하고 있어 릴리즈 번들에서 사용할 수 없다. `--wasm-dir` 인자를 추가하면 개발 환경(기본값)과 릴리즈 환경(`../filters`) 모두를 같은 바이너리로 지원할 수 있다.

매핑 규칙 (ADR-0004):
- nginx가 `nginx/` 디렉토리를 cwd로 실행되므로, Wasm 경로는 nginx prefix 기준 상대 경로
- 개발 환경 기본값: `../../target/wasm32-wasip1/wasm-release`
- 릴리즈 번들 환경: `../filters`

## Acceptance criteria

- [ ] `template::render(cfg, wasm_dir)` 시그니처가 `wasm_dir: &str` 인자를 받는다
- [ ] `gw-admin start`, `reload`, `generate` 서브커맨드에 `--wasm-dir` 인자가 추가된다 (기본값: `../../target/wasm32-wasip1/wasm-release`)
- [ ] `gw-admin generate --wasm-dir ../filters` 출력의 wasm 블록이 `../filters/api_key.wasm` 형태의 경로를 포함한다
- [ ] 기본값으로 실행 시 기존 통합 테스트 12/12가 유지된다
- [ ] `template.rs` 유닛 테스트가 `wasm_dir` 파라미터를 검증한다

## Blocked by

None — 즉시 시작 가능

**Desired behavior:**
`pub fn render(cfg: &GatewayConfig, wasm_dir: &str) -> String`으로 시그니처를 변경한다. `wasm_dir`은 nginx prefix(cwd) 기준 상대 경로이며, wasm 블록을 동적으로 생성한다:

```

    module rate_limit_filter          <wasm_dir>/rate_limit.wasm;
    module api_key_filter             <wasm_dir>/api_key.wasm;
    module header_manipulation_filter <wasm_dir>/header_manipulation.wasm;
    module logging_filter             <wasm_dir>/logging.wasm;
    module passthrough_filter         <wasm_dir>/passthrough.wasm;
}
```

`gw-admin start/reload/generate` 세 서브커맨드에 `--wasm-dir <path>` 인자를 추가한다. 기본값은 `../../target/wasm32-wasip1/wasm-release`로 개발 환경과 호환성을 유지한다.

- `pub fn render(cfg: &GatewayConfig, wasm_dir: &str) -> String`

- `gw-admin reload --wasm-dir <path>`
