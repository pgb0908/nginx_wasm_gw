## Agent Brief

**Category:** bug
**Summary:** wasmtime 샌드박스가 파일 접근을 막아 api-key·rate-limit·header-manipulation 필터가 시작 시 config 로드에 실패하고 모든 요청이 401/빈 응답을 반환한다

**Current behavior:**
세 필터의 `on_configure`가 `get_plugin_configuration()`으로 파일 경로를 읽은 뒤 `std::fs::read_to_string`으로 파일을 열려 한다. ngx_wasm_module의 wasmtime 샌드박스가 WASI filesystem preopen을 허용하지 않아 파일 읽기가 실패하고 경고 로그만 남긴다. 결과적으로 allowed key 목록이 비어 모든 요청이 401, rate limit config가 없어 즉시 429, header 조작이 일어나지 않는다. 통합 테스트 4/12 통과.

**Desired behavior:**
세 필터의 `on_configure`가 `get_plugin_configuration()`으로 JSON 문자열을 직접 받아 파싱한다. nginx.conf의 `proxy_wasm` 지시어가 인라인 JSON config를 포함한다. 통합 테스트 12/12 통과.

**Key interfaces:**
- `ApiKeyRoot::on_configure` — `get_plugin_configuration()`로 받은 바이트를 `parse_api_keys()`에 직접 전달 (파일 경로 경유 없음)
- `RateLimitRoot::on_configure` — 동일하게 `parse_config()`에 직접 전달
- `HeaderManipulationRoot::on_configure` — 동일하게 `HeaderManipulationConfig::parse()`에 직접 전달
- nginx.conf `proxy_wasm` 지시어 — `'<json>'` 인라인 포맷 복원. 현재 `gateway/config/policies/` 리소스 파일의 값을 기준으로 작성

**Acceptance criteria:**
- [ ] api-key 필터가 `get_plugin_configuration()`의 JSON에서 키 목록을 파싱한다
- [ ] rate-limit 필터가 `get_plugin_configuration()`의 JSON에서 `requests_per_second`를 파싱한다
- [ ] header-manipulation 필터가 `get_plugin_configuration()`의 JSON에서 add/remove 규칙을 파싱한다
- [ ] nginx.conf 각 location 블록에 인라인 JSON config가 포함된다
- [ ] `cargo test --workspace` 전체 통과
- [ ] `just integration` 12/12 통과

**Out of scope:**
- Admin Process 구현 (이슈 #08–#11)
- Resource Model JSON 파일에서 nginx.conf를 자동 생성하는 로직
