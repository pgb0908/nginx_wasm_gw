# 13 — `just release` 번들 생성

## What to build

`just release`가 self-contained `tar.gz` 릴리즈 번들을 로컬에서 생성한다. 버전은 현재 git tag에서 읽으며, 태그가 없으면 `v0.0.0-dev`로 fallback한다. 번들 구조는 ADR-0004를 따른다.

생성되는 번들:
```
nginx-wasm-gw-<version>-linux-x86_64.tar.gz
└── nginx-wasm-gw/
    ├── bin/
    │   ├── gw-admin          ← cargo build --release
    │   └── nginx             ← gateway/bin/nginx 복사
    ├── filters/              ← *.wasm (wasm-release 프로파일)
    ├── nginx/                ← nginx prefix (cwd)
    │   ├── logs/
    │   └── tmp/{client_body,proxy,fastcgi,scgi,uwsgi}/
    ├── config/               ← gateway/config/ 예시 파일 복사
    └── README.md
```

번들 안에서의 실행 방법: `./bin/gw-admin start --wasm-dir ../filters`

## Acceptance criteria

- [ ] `just release` 실행 시 `nginx-wasm-gw-<version>-linux-x86_64.tar.gz`가 생성된다
- [ ] git tag 없을 때 버전이 `v0.0.0-dev`로 fallback된다
- [ ] 번들 내 `bin/gw-admin`, `bin/nginx`, `filters/*.wasm`, `config/`, `nginx/logs/`, `nginx/tmp/` 디렉토리가 모두 존재한다
- [ ] 번들을 압축 해제한 뒤 `./bin/gw-admin start --wasm-dir ../filters`로 nginx가 정상 기동된다
- [ ] `gw-admin`은 `--release` 프로파일로 빌드된다 (개발용 debug 바이너리가 아님)

## Blocked by

- 12-wasm-dir-parameter.md (`--wasm-dir ../filters` 지원이 완료되어야 번들에서 사용 가능)

```
2. Wasm 필터 빌드
   cargo build --workspace --target wasm32-wasip1 --profile wasm-release --exclude gateway-admin

3. gw-admin 릴리즈 빌드
   cargo build -p gateway-admin --release

4. 번들 디렉토리 조립
   nginx-wasm-gw-<VERSION>-linux-x86_64/
     bin/gw-admin   ← target/release/gw-admin

     nginx/logs/
     nginx/tmp/{client_body,proxy,fastcgi,scgi,uwsgi}/
     config/        ← gateway/config/ 전체 복사
     README.md

5. tar.gz 생성 + 디렉토리 정리
   nginx-wasm-gw-<VERSION>-linux-x86_64.tar.gz
```
