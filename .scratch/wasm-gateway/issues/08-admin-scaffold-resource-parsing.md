# 08 — Admin Process 뼈대 + Resource Model 파싱

## What to build

`gateway/admin/` Rust 크레이트를 생성하고 `gw-admin` 바이너리의 CLI 뼈대를 구현한다. `gateway/config/<kind>/` 디렉토리에서 Listener/Router/Service/Policy JSON 파일을 읽어 Rust 구조체로 파싱하는 로직을 구현한다.

리소스 모델은 Kubernetes 스타일(`apiVersion: iip.gateway/v1alpha1`, `kind`, `metadata`, `spec`)을 따르며 스펙은 `docs/config-models/`에 정의돼 있다. Policy의 `policyRef` 필드는 제거됐다 (ADR-0003).

- [ ] `gw-admin` 바이너리가 `start`, `stop`, `reload`, `status` 서브커맨드를 갖는다 (미구현 서브커맨드는 "not yet implemented" 출력)
- [ ] `gateway/config/listeners/`, `routers/`, `services/`, `policies/` 디렉토리의 JSON 파일을 읽어 Rust 구조체로 파싱한다
- [ ] 잘못된 JSON 또는 알 수 없는 `kind`에 대해 명확한 에러 메시지를 출력한다
- [ ] 파싱 로직에 대한 유닛 테스트가 존재한다

## Agent Brief

**Category:** enhancement
**Summary:** `gateway/admin/` Rust 크레이트를 생성하고 `gateway/config/<kind>/` 디렉토리의 Resource Model JSON을 Rust 구조체로 파싱한다

**Current behavior:**
Admin Process가 존재하지 않는다. `gateway/config/` 하위 리소스 파일들(Listener/Router/Service/Policy JSON)이 작성돼 있으나 이를 읽는 코드가 없다.

**Desired behavior:**
`gw-admin` 바이너리가 `start`, `stop`, `reload`, `status` 서브커맨드를 갖는다. `gateway/config/listeners/`, `routers/`, `services/`, `policies/` 디렉토리를 스캔해 각 JSON 파일을 `Listener`, `Router`, `Service`, `Policy` Rust 구조체로 파싱한다. 리소스 스펙은 `docs/config-models/`에 정의돼 있다. `policyRef` 필드는 스키마에서 제거됐으므로 무시한다.

**Key interfaces:**
- `GatewayConfig` — 파싱된 전체 리소스를 담는 최상위 타입. `listeners: Vec<Listener>`, `routers: Vec<Router>`, `services: Vec<Service>`, `policies: Vec<Policy>` 필드
- `ResourceKind` enum — `Listener`, `Router`, `Service`, `Policy` 구분
- `Policy.spec.order: u32` — 실행 순서. 낮을수록 먼저 실행
- `Policy.spec.target_ref.name` — `"*"`이면 모든 Router에 적용
- `PolicyConfig` enum — `Security(SecurityConfig)`, `Traffic(TrafficConfig)`, `Transform(TransformConfig)`, `Enhance(EnhanceConfig)` 구분. `SecurityConfig`는 `api_key: Option<ApiKeyConfig>` 포함
- CLI: `clap` 크레이트 권장

**Acceptance criteria:**
- [ ] `gateway/admin/` 크레이트가 workspace `Cargo.toml`에 등록된다
- [ ] `gw-admin --help`가 `start`, `stop`, `reload`, `status` 서브커맨드를 나열한다
- [ ] `gateway/config/` 현재 파일들(listener 1개, router 2개, service 2개, policy 3개)이 에러 없이 파싱된다
- [ ] 알 수 없는 `kind` 또는 필수 필드 누락 시 명확한 에러 메시지를 출력하고 non-zero exit
- [ ] `PolicyConfig` 타입이 Security/Traffic/Transform을 구분한다
- [ ] 파싱 로직에 대한 유닛 테스트가 존재한다

**Out of scope:**
- nginx.conf 생성 (이슈 #09)
- nginx 프로세스 관리 (이슈 #10)
- 파일 감시 (이슈 #11)
